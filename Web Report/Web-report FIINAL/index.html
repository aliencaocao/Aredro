Problem

What is the problem found?

access_time

Slow

We found out that the councillors needed to spend a long time to bring initiatives to the students, especially food ordering

sentiment_very_dissatisfied

Inconvienient

We found out that the iEMB is very inconvienient for the students to enjoy the initiatives that the councillors bring out.

For example, if they were to order some food from the council, they will have to login to iEMB and fill up a Google Form.

What is another weakness of iEMB? It does not provide the best experience on mobile devices and sometimes it will be harder to operate on these devices.

Objectives

What do we want to achieve using Aredro?

timer

Save time

An objective of our project is to help both students and councillors to save time in ordering/bringing initiatives. Time is very precious in HCI, hence it is necessary to save as much time as we can.
More information on our solution can be found below.

event_available

Convenience

The app also aims to be as easy and convenient as possible. iEMB is not so convenient and effecient when it comes to ordering. The use of Google Forms also reduce effeciency.
Aredro focusses on ordering and aims to make lives of HCI students easier, espically councillors'.

Features

Hence, we came up with this idea of creating an app that allow users to order on the go.

It has the following key features.

flash_on

Faster

According to our comparisons, the councillors will save 90% of their time if they were to use Aredro to sell items their items instead of iEMB, resulting in more efficent ordering.

How?

group

User System

This app can be accessed by two parties: managers(councillors) and students. They have different privileges in the app.

  • Managers - The users are usually councillors, developers and sometimes testers. These people are able to add orders, check orders and remove orders.

  • Students - The users are usually students and sometimes testers. These people are able to check and submit orders.

lightbulb_outline

Easier

The UI was designed to be easy to use. There are no complexed layouts on the screen. Beginners can also get help easily.

Learn more

Material Design

Our app is unique. It achieved what seems to be impossible in an App Inventor app, which is Google Material Design.
These differences and unique features proved our passion into the development of Aredro.

menu

Side Bar

The app has a side navagation bar just like any mordern apps, but unlike any App Inventor apps.
It is a place to keep all functions easily accessible, preventing overflow of buttons on the screen. All functions are accompanied by an icon, made by Google Inc. Click for example.

apps

Button

The buttons in the app are very different from normal App Inventor apps. It has the " ripple" effect which is a part of Material Design. It aims to improve the responsiveness of the app and provide better UX.
This is not possible at all in App Inventor without any special tools. However, we managed to do it with help of third-party tools and self-made extensions.
Compare between the AI2 button and the Material button

create

Text Box

The text boxes in the app are different from App Inventor's default style. The default style is not visually appealing as it occupies a lot of screen space.
The new Material style is much simpler and looks tidier. (Click to view examples)
We did some modding to the app and changed the style to fit Material Design.

timelapse

Loading

The app uses a new loading dialogue instead of the default one. The style also features Material Design and is present in all mordern apps.

Functions

Here is a summary of all major functions of the app.
Screenshots of each function can be found below.

account_circle

Register & Login

The app requires users to register an account before using.
Registration is to differenciate Manager and Student accounts. A manager account cannot login as a student unless the same username was also registered as student account, vice versa.
Refer to the "Account System" section in the "features" section above for more information.

attach_money

Order

The app allows student users to order items by filling up their particulars. All submitted orders will go to the Managers and only they can view the order list.
The app features a dynamic confirmation message before anything is submitted.

view_list

Check Item

Both students and managers can access the "check items" function. Students can use it to check avilable items to order and make their decision based on the list.
There may be multiple manager accounts. Managers use this function to check what item has been added to the list so that duplicated ones will not be added.

add_circle

Add Item

This function allows managers to add items for sale by filling up necessary information about the item. E.g. Price, choices available etc.
All added items will be compiled into a list and both students and managers would be able to access.
Students will place their orders based on the list.

done_all

Check Orders

It allows managers to view a list of orders received from students so that the Managers may prepare/process the orders.

delete_sweep

Delete Item

It allows managers to delete overdued items for sale so that they can add a new array of items when neccessary. Deleted items will no longer appear in the item list displayed to students, to prevent them from getting confused on which is available.

highlight_off

Delete Orders

It allows managers to delete received orders once they have processed all of them, so that when new orders came in, they will not be confused on which to attend to.

Limitations

What we cannot achieve so far?

iOS

The app currently can only run on Android devices. We plan to build the iOS version for Apple users in the school if we are continuing.

* The Apple logo and its name are registered trade marks of Apple Inc.

shop

Play Store

The app currently is not downloadable from Play Store, but only from our website. We plan to publish it on Play Store if it is being used in school so that it eases the process of downloading and updating.

notifications

Notification

The app currently does not alert the user when the ordered item is ready/a new item has just been added for sale. We plan to add in these notifications in the future.
